# Qard — полировка к релизу (дизайн-спека)

Дата: 2026-06-29
Статус: одобрено пользователем

## Контекст

Qard — приложение для изучения языка (React + TypeScript, две сборки: Electron
через `window.electronAPI` и web). Состояние — zustand (`src/renderer/store`).
Текущий режим обучения: английский (определение на английском, пользователь
вводит английское слово). Локализации в проекте нет — все строки хардкод на
английском.

**Две независимые палитры:**
- Глобальная «графитовая» — токены в `src/renderer/App.css :root`.
- Неоново-синяя — локально объявлена внутри `.review` (`Review.css:1-22`),
  переопределяет глобальную в зоне Review.

Любое тематическое изменение должно учитывать ОБА набора токенов.

**Шапка** (`App.tsx` / `App.css .app-header`) — grid `1fr auto 1fr`:
бренд (слева) | навигация (центр) | **пустая правая колонка**. Правая колонка —
штатное место под управляющие тумблеры.

## Общий принцип

UI-настройки (тема, язык, звук) — клиентские preferences, не бизнес-данные.
Хранятся в `localStorage`, а НЕ через `electronAPI.settings` IPC — чтобы работать
одинаково в web и Electron без правки IPC-хендлеров. Доступ через лёгкий React-контекст.

## Фичи

### 1. Кастомный анимированный Select

Переписать `src/renderer/components/common/Select.tsx` с нативного `<select>` на
доступный компонент: кнопка-триггер + выпадающий список `role="listbox"`.

- **Анимация раскрытия:** `opacity` + `transform: translateY/scaleY`, transition
  ~160ms ease-out; закрытие чуть быстрее. Не использовать прыгающий `max-height`.
- **Доступность:** навигация стрелками вверх/вниз, Enter (выбор), Esc (закрыть),
  корректные `aria-*` (`aria-expanded`, `role="listbox"`/`role="option"`,
  `aria-selected`). Клик вне компонента закрывает.
- **Дизайн:** графитовые токены (`--surface`, `--border`, `--accent`).
- **Совместимость:** сохранить публичный API `options: {value,label}[]`, `value`,
  `label`, `error`. Сигнатура `onChange` меняется с DOM-event на `(value: string)`.
  Обновить 3 места вызова: Browser.tsx, Preview.tsx, Settings.tsx.

**Критерий приёмки:** дропдаун открывается/закрывается плавно, управляется с
клавиатуры, выбор значения работает во всех трёх экранах, визуально совпадает с темой.

### 2. Отступ над «Current Settings»

Точечная правка `src/renderer/components/Generation/Generation.css`: уменьшить
верхний `margin` блока `.settings-summary` (и при необходимости нижний у
`.description`).

**Критерий приёмки:** блок «Current Settings» визуально ближе к заголовку, без
лишнего пустого пространства.

### 3. i18n RU/EN (только интерфейс)

Без сторонних библиотек. Своё лёгкое решение:

- `src/renderer/i18n/` — словари `ru.ts` / `en.ts` (плоские ключи) +
  `I18nProvider` (React context) с хелпером `t(key)`.
- Текущий язык — в контексте + `localStorage` (ключ `qard:lang`).
- **Дефолт — RU** (аудитория РФ).
- Извлечь хардкод-строки в ключи из: App, Decks, Browser, Settings, Generation,
  WordInput, Review, общих модалок (ConfirmModal, CreateDeckModal, ImportModal,
  CardEditModal), Preview.
- Переключатель **RU/EN — сегмент-тумблер в правой колонке шапки** (`justify-self:end`).

Направление изучения НЕ меняется — это только язык интерфейса. Структура словарей
закладывается с прицелом на будущую мультиязычность.

**Критерий приёмки:** переключение RU/EN мгновенно меняет все подписи интерфейса;
выбор сохраняется между сессиями; нет оставшихся хардкод-строк в перечисленных
компонентах.

### 4. Звуки ответа (Web Audio синтез)

- `src/renderer/audio/sfx.ts` — один лениво-инициализируемый `AudioContext`
  (по первому пользовательскому жесту), функции `playCorrect()` / `playWrong()`.
  Огибающая gain: быстрый attack, плавный release ~150-250мс; тихая громкость
  (~0.15). Correct = две восходящие ноты, wrong = одна мягкая низкая.
- Вызов в `Review.submit()` по результату `isCorrect`.
- **Mute-тумблер** (иконка-динамик) в правой колонке шапки; состояние в
  `localStorage` (ключ `qard:sound`).

**Критерий приёмки:** при верном ответе — приятный короткий звук вверх, при
неверном — мягкий низкий; mute полностью отключает; громкость не раздражает.

### 5. Светлая тема + тумблер

- Атрибут `data-theme="light|dark"` на корневом элементе.
- Светлая палитра — переопределение CSS-переменных для ОБОИХ наборов: глобального
  `:root` и `.review` (мягкий тёплый светлый фон; неон → приглушённый акцент).
- Тумблер (иконка солнце/луна) в правой колонке шапки рядом с RU/EN.
- Дефолт — тёмная (текущая). Сохранение в `localStorage` (ключ `qard:theme`).

**Критерий приёмки:** переключение темы мгновенно перекрашивает и оболочку, и
экран Review; светлая тема читаема и не «кислотная»; выбор сохраняется.

### 6. Слабее свечение карточки

В `Review.css`, `.review-capsule`:
- Убрать/сильно замедлить бесконечную анимацию `neon-pulse` (источник
  отвлекающей пульсации).
- Уменьшить blur/spread и opacity внешних слоёв `box-shadow`
  (`--neon-soft` 0.55 → ~0.3; внешний `90px`-слой убрать или приглушить).
- `.is-correct` / `.is-wrong` оставить как короткую мягкую вспышку-обратную-связь.

**Критерий приёмки:** рамка карточки в покое не пульсирует и не отвлекает;
обратная связь на ответ всё ещё заметна.

### 7. Ограничение поля ввода ответа (английский + лимит длины)

Замечание по сути верное (ответ — английское слово), но реализация «в лоб» вредна.
Профессиональный подход в `.review-input` (`Review.tsx`):

- **Не блокировать нажатия клавиш жёстко.** Мягкая нормализация на `onChange`:
  отфильтровывать не-латиницу, но **разрешить пробел, дефис, апостроф**
  (для «don't», «well-being», фраз).
- **Лимит длины:** через нативный `maxLength`. Значение — длина самого длинного
  слова в текущей колоде + запас, либо разумный потолок ~40.
- **Обратная связь:** при попытке ввести кириллицу — не молча съедать символ, а
  кратко мигнуть полем / показать хинт «English only», чтобы не воспринималось багом.

**Критерий приёмки:** в поле нельзя ввести кириллицу; пробел/дефис/апостроф
работают; длина ограничена; при вводе кириллицы есть видимая реакция.

## Размещение управляющих элементов

Правая колонка шапки (`justify-self:end`), слева направо:
`[тема ☀/☾] [RU/EN] [звук 🔊/🔇]` — три компактных контрола.

## Что НЕ делаем (YAGNI)

- Не меняем направление изучения (только язык интерфейса).
- Не подключаем i18n-библиотеки.
- Не используем звуковые файлы (только синтез).
- Не трогаем IPC-хендлеры ради UI-преференсов.
