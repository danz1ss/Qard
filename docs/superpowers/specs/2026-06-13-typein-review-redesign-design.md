# Type-in Review с премиум-капсулой — Design Spec

> Дата: 2026-06-13. Область: только экран изучения (Review). Ветка: `feat/anki-functionality`.

## Цель

Переделать экран изучения из режима «слово → определение + 4 кнопки самооценки» в **активное припоминание с вводом слова** (type-in) и премиум-визуал: тёмно-синий фон, центральная капсула с голубым неоновым свечением и виньеткой, плавные анимации. Пользователь читает определение и пример с пропуском, вписывает загаданное слово, жмёт Enter и сразу видит верно/неверно; затем раскрывается полная карточка.

## Поведение

### Бинарный режим
- Убираются кнопки **Трудно / Хорошо / Легко**. Остаётся только **верно / неверно**, определяемое сравнением ввода со словом карточки.
- Один ввод за карточку. Нет режима «отвечай пока не угадаешь»: после Enter показывается результат и раскрытая карточка, дальше — следующая.

### Маппинг в FSRS
- **Верно → Good (rating 3)**.
- **Неверно → Again (rating 1)**.
- FSRS-логика, очередь и дневные лимиты НЕ меняются. Это даёт желаемое поведение: недавно изученное всплывает в ближайших сессиях (карточка встречается ~2 раза за счёт learning-steps), затем интервалы растут; неверный ответ возвращает карточку раньше, но не зацикливает сессию.
- IPC `review.answer(cardId, rating)` и `previewIntervals` уже существуют и используются как есть (previewIntervals на фронте больше не показывается — оценка автоматическая).

### Проверка ответа
- `ввод.trim().toLowerCase() === card.word.trim().toLowerCase()`.
- Регистр и крайние пробелы игнорируются. Опечатки/словоформы НЕ засчитываются (точное совпадение) — это осознанный выбор (YAGNI; при необходимости смягчим позже).

## Поток экрана

```
СОСТОЯНИЕ 1 — вопрос (до ввода).
Капсула содержит определение + пример; строка ввода — ОТДЕЛЬНО под капсулой.

  ╭─────────────── капсула (неон) ───────────────╮
  │  ОПРЕДЕЛЕНИЕ        ← мини-метка (приглушённый │
  │  A ______ is the planet we live on…   голубой)│
  │                                                │
  │  ПРИМЕР             ← мини-метка                │
  │  She travels around the ______ every year.     │
  ╰────────────────────────────────────────────────╯

  ┌─────────────────────────────┐
  │ впиши слово…            ▌    │   ← светящаяся строка ввода (autofocus), под капсулой
  └─────────────────────────────┘
        Enter — проверить

СОСТОЯНИЕ 2 — результат (после Enter):
  • Капсула и строка ввода подсвечиваются: зелёный неон (верно) / красный неон (неверно).
  • При неверном — лёгкий shake строки ввода.
  • Раскрывается полная карточка (для обоих исходов, чтобы запомнить правильный ответ):
      world            ← загаданное слово, крупно
      /wɜːld/          ← транскрипция
      ▶ Аудио          ← кнопка + автоплей при раскрытии
      ОПРЕДЕЛЕНИЕ: …   ← определение целиком (слово показано)
      ПРИМЕРЫ:
        • She travels around the **world** every year.   ← слово выделено жирным
        • The **world** is a beautiful place.
  • Enter / пробел / кнопка «Дальше» → review.answer(rating) → следующая карточка.
```

### Содержимое на фронте
- **Определение**: поле `definition` (генерация уже кладёт в него `______` вместо слова). Показывается как есть под меткой «ОПРЕДЕЛЕНИЕ».
- **Пример**: поле `definitionExample` (одно предложение). Загаданное слово в нём обёрнуто в `<b>…</b>` генерацией — на фронте эта обёртка **заменяется на `______`**. Показывается под меткой «ПРИМЕР».
  - Fallback: если в `definitionExample` нет `<b>`-обёртки (или поле пустое), блок «ПРИМЕР» не показывается на фронте (остаётся только определение + ввод).
- Массив `examples` на фронте **не показывается** (раскрывается только в результате).

### Содержимое в результате
- Слово (`word`), транскрипция (`transcription`), аудио (если есть), определение целиком, все `examples`.
- В примерах и в раскрытом определении `<b>…</b>` рендерится как **жирный текст** (а не сырые теги).

## Обработка контента (`<b>`-теги)

Маленький хелпер в renderer:
- `blankOut(html, placeholder='______')` — заменяет `<b>…</b>` (и формы внутри) на пропуск, остальной HTML убирает. Используется для примера на фронте.
- `renderBold(html)` — превращает `<b>…</b>` в `<strong>`, остальные теги убирает; рендерится безопасно (без сырого пользовательского HTML — только распознанные `<b>`/`<br>`). Используется в результате.
- Никакого `dangerouslySetInnerHTML` с непроверенным содержимым: парсим в массив React-узлов (текст + `<strong>`), экранируя всё прочее.

## Визуальный дизайн (только Review)

- **Фон**: тёмно-синий/графитовый (не чистый чёрный), радиальный градиент от центра + лёгкая виньетка по краям экрана.
- **Капсула**: сильно скруглённый контейнер (pill/rounded-rect), голубое неоновое свечение по контуру (многослойный `box-shadow`), внутренняя виньетка (radial-gradient затемнение к краям). Контур тонкий, светящийся.
- **Акцент**: голубой неон (cyan/azure) — свечение капсулы, фокус строки ввода, метки.
- **Состояния результата**: зелёный неон (верно), красный неон (неверно) — перекрашивают свечение капсулы и рамку ввода.
- **Метки «ОПРЕДЕЛЕНИЕ»/«ПРИМЕР»**: мелкий uppercase, letter-spacing, приглушённый голубой.
- **Шрифт**: подключить премиум geometric sans (Manrope или Sora) локально через webpack/CSS; fallback на системный.
- **Анимации** (CSS, без тяжёлых библиотек):
  - появление карточки: fade + лёгкий scale-up;
  - постоянный мягкий пульс неонового свечения капсулы;
  - переход вопрос→результат: плавная смена цвета свечения;
  - неверный ответ: короткий shake строки ввода;
  - раскрытие нижнего блока: fade/slide-down.
  - Учитывать `prefers-reduced-motion` — отключать анимации.

## Затрагиваемые файлы

- `src/renderer/components/Review/Review.tsx` — переписать поток (type-in вместо reveal+4 кнопки): строка ввода, проверка, состояние результата, раскрытие, хоткеи (Enter — проверить/дальше).
- `src/renderer/components/Review/Review.css` — премиум-редизайн (тёмная тема, капсула, неон, анимации, метки, шрифт).
- `src/renderer/components/Review/htmlText.ts` (новый) — хелперы `blankOut` / `renderBold` с юнит-тестами.
- Возможный шрифт-ассет + правило `@font-face` (или импорт пакета шрифта).
- Генерация (`gemini.service.ts`) НЕ меняется: `definition` уже с пропуском, примеры уже с `<b>`. Промпт не трогаем.

## Тестирование

- Юнит (vitest) для `htmlText.ts`: `blankOut` заменяет `<b>word</b>` на `______`, убирает прочий HTML, корректно работает без `<b>`; `renderBold` даёт узлы с `<strong>` для `<b>` и экранирует остальное. Без сырого HTML-инъекшна.
- Проверка ответа (можно вынести чистую функцию `isCorrect(input, word)` и покрыть): регистр/пробелы игнорируются, точное совпадение.
- Ручной сквозной: Decks → «Учить» → ввод верного слова (зелёный, раскрытие, аудио), ввод неверного (красный, shake, раскрытие), Enter→следующая, пустая очередь, `prefers-reduced-motion`.
- Существующие 31 тест (сервисы/маппер) не затрагиваются и остаются зелёными.
