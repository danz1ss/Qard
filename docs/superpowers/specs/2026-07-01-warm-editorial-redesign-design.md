# Qard — тёплый editorial-редизайн, фаза 1 (дизайн-спека)

Дата: 2026-07-01
Статус: одобрено пользователем (визуальный брейншторм через mockup-компаньон)

## Контекст

Изначальный запрос описывал редизайн в духе языкового курса (уроки, quiz-карточки,
streak, достижения). По факту Qard — генератор Anki-карточек с ИИ + SRS-повторение,
без курсов/уроков в привычном смысле. Пользователь подтвердил: редизайним
**существующий продукт**, ничего не выдумываем и не добавляем новых фич геймификации
(п. "НЕ ВЫДУМЫВАЙ").

Приложение — React + TypeScript, общий код между Electron (`window.electronAPI`)
и web-версией (`src/web`, freemium free-tier — см. [[free-tier-web]]). 6 вкладок:
Decks, Browse, Setup, Input, Generate, Preview (`src/renderer/App.tsx`).

**Текущее состояние дизайна** (после недавно влитой ветки release-polish,
2026-06-30):
- Глобальная тёмно-графитовая тема — токены в `src/renderer/App.css :root`
  (default = тёмная), светлая тема — `:root[data-theme='light']` override.
- Экран Review (`Review.css`) держит **отдельную неоново-синюю палитру**
  (`--bg-0`, `--neon`, `--ok`, `--bad` + свой light-override) — визуально
  ощущается как "другой продукт" внутри приложения.
- Шрифт везде — Rubik (400/500/600/700). Отдельного premium-шрифта (Sora) в коде
  нет — это не подтвердилось, работаем только с Rubik.
- Проект уже большой и полирован (i18n RU/EN, темы, звуки, кастомный Select) —
  редизайн должен быть эволюцией токенов, а не поломкой существующей архитектуры.

Полный охват — "каждая вкладка" — слишком большой объём для одного цикла
план→реализация→проверка. Разбиваем на фазы:

- **Фаза 1 (эта спека):** дизайн-токены (обе темы) + Decks + Review.
- **Фазы 2+ (отдельные спеки позже):** Browse, Setup, Input, Generate, Preview —
  их собственная структура/карточки/отступы. Они автоматически получат новую
  палитру через глобальные токены и общие компоненты (см. ниже), но не
  структурный редизайн — это последующие итерации.

## Общий принцип

**Направление:** тёплый editorial-минимализм.
- Светлая тема (основная): кремовый фон, почти белые карточки, мягкие тени
  вместо градиентов.
- Тёмная тема: тёплый тёмно-шоколадный фон (не холодный графит), тот же язык.
- **Акцент — глубокий индиго** (осознанный холодный контраст к тёплой базе,
  выбран пользователем после сравнения с золотом/бордо/оливковым).
- Статусные цвета SRS (New/Learning/Due) остаются смысловыми и отделены от
  акцента: teal / охра / лесной зелёный.
- Типографика — **только Rubik** везде, без дополнительных шрифтов.
- Никакой Tailwind/новых зависимостей — те же CSS-переменные и `.css`-файлы
  рядом с компонентами, что и сейчас.
- Никакой новой геймификации — `streak`/`dailyGoal` остаются как есть
  (данные и логика не трогаются), меняется только их визуальное оформление.

**Важно про фазировку:** `Button`, `Input`, `Select`, `Modal` — общие компоненты,
использующиеся всеми 6 вкладками. Меняя их токены в App.css, мы автоматически
обновляем базовый вид (фон, кнопки, инпуты, шапку) во всём приложении сразу —
поэтому после фазы 1 экраны вне охвата (Setup, Browse, Generate, Input, Preview)
не будут выглядеть "старой графитовой заплаткой", а получат новую базовую
палитру. Их собственная структурная перекомпоновка — уже в следующих фазах.

## Дизайн-токены (`src/renderer/App.css :root`)

### Светлая тема (заменяет текущий `:root[data-theme='light']`)

```
--bg: #f4efe6;        --bg-deep: #ece5d8;    --bg-elev: #fffdf9;
--surface: #fffdf9;   --surface-hi: #fbf6ee; --border: #e4dcc9;
--border-soft: #ece5d5;
--ivory (text-primary): #2b2620;
--ivory-dim (text-secondary): #6b6154;
--muted: #8a8071;
--accent: #3a4a6b;    --accent-hi: #4d5f85;  --accent-press: #2c3852;
--accent-soft: rgba(58, 74, 107, 0.12);
--new: #3d6b6e;  --learn: #b5793a;  --due: #4d6e50;  (--green:=--due, --amber:=--learn)
```

`--coral`/`--coral-shade` (уже существующий глобальный токен ошибок/danger —
используется в Input, Select, Button, Generation, Settings, Preview)
переиспользуется как есть для "неверного ответа" в Review — он уже тёплый и
подходит палитре, отдельная `--bad`-переменная не нужна.

### Тёмная тема (заменяет текущий default `:root`)

```
--bg: #201c17;        --bg-deep: #1a1713;    --bg-elev: #2a251e;
--surface: #2a251e;   --surface-hi: #332c23; --border: #423a2e;
--border-soft: #3a332a;
--ivory: #f4efe6;     --ivory-dim: #b9ac97;  --muted: #8a7d6a;
--accent: #7b8fc4;    --accent-hi: #93a4d1;  --accent-press: #64769e;
--accent-soft: rgba(123, 143, 196, 0.16);
--new: #5fa3a6;  --learn: #dba458;  --due: #7bbf85;
```

`--coral`/`--coral-shade` в тёмной теме остаются как есть (уже тёплые:
`#e8856e`/`#b85c48`) — используются для "неверного ответа" в Review, как и во
всех остальных экранах.

Существующие имена переменных (`--ivory`, `--muted`, `--green`, `--amber` и т.д.)
сохраняются один в один, меняются только значения — чтобы не трогать все места
использования в остальных 5 вкладках без необходимости.

### Типографика

Единая шкала (добавляется в App.css, применяется к затронутым файлам):

```
--text-xs: 12px/1.4    --text-sm: 14px/1.5   --text-base: 16px/1.6
--text-md: 18px/1.5    --text-lg: 22px/1.35  --text-xl: 28px/1.25
--text-2xl: 36px/1.2
--font: 'Rubik', system-ui, -apple-system, sans-serif;  (без изменений)
```

### Spacing / Radius / Motion

- Отступы: существующая шкала 4/8/12/16/24/32/48 + добавляется `--space-8: 64px`.
- Радиусы: `--radius: 16px` (карточки, как сейчас), `--radius-sm: 10px` (инпуты),
  `--radius-lg: 24px` (капсула Review, крупные hero-блоки).
- Моушен: `150–250ms ease` — уже используется в проекте, сохраняется как норма
  для новых transition/hover-состояний.

## Затронутые файлы (фаза 1)

- `src/renderer/App.css` — токены обеих тем, шапка/навбар, глобальные `:where()`
  примитивы (button/input/select).
- `src/renderer/components/common/{Button,Input,Select,Modal,ConfirmModal,
  CreateDeckModal}.css` — цвета/тени под новые токены (без изменения логики/API).
- `src/renderer/components/Decks/{Decks.css, StudyHeader.css}` — карточки колод,
  онбординг, поповер лимитов, стрик/кольцо/недельные бары.
- `src/renderer/components/Review/Review.css` — **удаляются локальные неоновые
  токены** (`--bg-0`, `--bg-1`, `--neon*`, `--ok*`, `--bad*` и их
  `:root[data-theme='light'] .review` override), капсула переводится на общие
  токены (`--surface`, `--border`, `--accent`, `--due`, `--coral`).

Логика компонентов (`.tsx`) не меняется — только `.css`. Исключение: если при
переносе капсулы на общие токены понадобится убрать теперь неиспользуемые
inline-стили/классы — точечно, без изменения поведения (fetch очереди, ввод,
аудио, счётчик промахов и т.д. остаются как есть).

## Фичи

### 1. Глобальные токены — тёплая палитра (обе темы)

Переписать оба блока токенов в App.css значениями выше. Проверить, что везде,
где раньше был жёсткий hex вместо переменной (например неон в Review), теперь
используется общий токен.

**Критерий приёмки:** в App.css не осталось графитовых hex-значений; обе темы
переключаются мгновенно (как сейчас) и дают тёплую светлую/тёплую тёмную
палитру; контраст текста к фону — не хуже текущего (AA для основного текста).

### 2. Единая типографика на Rubik

Применить шкалу `--text-*` к заголовкам/тексту в затронутых файлах
(`.tab-content h2`, `.brand-text h1`, `.decks-title`, капсула Review и т.д.).
Другие шрифты не подключаются и не используются.

**Критерий приёмки:** во всех затронутых файлах единственный `font-family` —
`var(--font)` (Rubik); визуальная иерархия заголовков стала чище (нет
разнобоя размеров типа 25px/31px/17px без системы).

### 3. Decks — карточки колод, StudyHeader, онбординг

Тёплые карточки (`--surface` фон, `--border` рамка, мягкая тень вместо
градиентных inset-теней), CTA "Учить" — акцентная индиго-кнопка. Кольцо
дневной цели, стрик-иконка, недельные бары, поповер лимитов, пустое
онбординг-состояние — та же структура и логика, новые токены + при
необходимости точечная доводка отступов под шкалу `--space-*`.

**Критерий приёмки:** визуально соответствует утверждённому мокапу
(`final-direction-v2`); stagger-анимация появления карточек сохранена;
существующие брейкпоинты (≤860px, ≤520px) работают без регрессий.

### 4. Review — капсула без неона, единый визуальный язык

Убрать неоновое свечение/text-shadow/local light-override. Капсула — карточка
на `--surface`/`--border`/мягкая тень; состояния `is-correct`/`is-wrong` меняют
цвет рамки и текста ответа на `--due`/`--coral` соответственно (без glow).
Инпут, подсказка, блок раскрытия ответа (`reveal-*`) — на общих токенах.

**Критерий приёмки:** капсула визуально единообразна с остальным приложением
в обеих темах; поведение не изменилось (фокус на инпуте, Enter/Space —
следующая карточка, автопроигрывание аудио при раскрытии, шейк при неверном
ответе — сохраняется, только без неонового свечения).

### 5. Общие компоненты (Button/Input/Select/Modal)

Первичная кнопка — акцентный индиго-фон, фокус-кольцо — `--accent-soft`.
Инпуты/Select — `--surface`/`--border`, фокус — акцентная обводка. Modal/
ConfirmModal/CreateDeckModal — фон `--surface`, тень мягче текущей.

**Критерий приёмки:** т.к. это общие компоненты, изменения автоматически
проявятся и на вкладках вне фазы 1 (Setup, Browse, Input, Generate, Preview) —
без визуальных регрессий и без изменения их структуры/логики.

## Тестирование / верификация

- `npm run build` (web и electron конфиги) — без ошибок.
- Ручная проверка Decks и Review в light/dark, desktop и web dev-сборке.
- `npm test` — существующие тесты (`htmlText`, `collection.stats` и т.д.) не
  завязаны на CSS, должны пройти без изменений как регресс-проверка, что
  логика не задета.
- Визуальная проверка контраста текст/фон в обеих темах (не хуже текущего).

## Вне рамок фазы 1 (Out of scope)

- Browse, Setup, Input, Generate, Preview — получают новую базовую палитру
  через общие токены/компоненты, но их собственная вёрстка/карточки/отступы —
  отдельные спеки в следующих итерациях.
- Никакой новой геймификации (курсы/уроки/achievements/уровни) — это не
  соответствует продукту и не входит в задачу.
- Мобильная адаптивность — используются существующие брейкпоинты, меняются
  только цвета/токены внутри них.
