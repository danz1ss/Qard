# Дизайн: Web-PWA порт Qard (v1, бесплатно)

**Дата:** 2026-06-18
**Статус:** на ревью
**Связано:** `2026-06-18-qard-gtm-launch-design.md` (это его тех-фундамент, Фаза 1)

## Цель

Запустить Qard как **web-PWA** на бесплатной инфраструктуре, данные локально у юзера, без подписок. Главный канал распространения (ноль-барьерная ссылка `qard.pages.dev`). Electron-сборка остаётся как есть, не трогаем.

## Решения (зафиксированы с Даньком)

- **Инфра:** Cloudflare всё-в-одном — Pages (хостинг статики) + Worker (AI-прокси). Бесплатный аккаунт.
- **AI-провайдер:** ProxyAPI (`proxyapi.ru`), OpenAI-совместимый. BYOK — ключ юзера хранится локально.
- **Хранилище:** IndexedDB (дамп sql.js-базы). Настройки — localStorage.
- **TTS:** Web Speech API (браузерный синтез, бесплатно). Google/ElevenLabs — позже.

## Архитектура

Чистое разделение уже есть: `src/renderer` (UI) общается только через контракт `window.electronAPI` (preload, 9 групп API). Порт = реализовать тот же контракт браузерным адаптером.

```
  ПЕРЕИСПОЛЬЗУЕМ как есть          ПИШЕМ ЗАНОВО (web-адаптер)
  ┌──────────────────────┐        ┌─────────────────────────────────┐
  │ renderer (весь UI)   │        │ window.qardAPI (тот же контракт): │
  │ CollectionService    │        │  collection/review/stats         │
  │  (вся sql.js-логика) │   +    │    → CollectionService + IndexedDB│
  │ FSRS scheduler       │        │  settings  → localStorage         │
  │ shared/types         │        │  ai        → fetch → CF Worker    │
  │ import.mapper        │        │  tts       → Web Speech API        │
  │ wordParser           │        │  shell     → window.open          │
  └──────────────────────┘        │  media/importer → см. «Отложено»  │
                                   └─────────────────────────────────┘
```

UI не меняется. Имя моста (`electronAPI`) либо переиспользуем, либо вводим общий `window.qardAPI` с двумя реализациями (electron/web) — решить в плане.

## Компоненты v1

### 1. Web-адаптер хранилища
- `CollectionService.init(data)` / `.export()` уже есть. Заменяем файловый слой (`collection.storage.ts`) на браузерный: загрузка дампа из IndexedDB при старте, дебаунс-сохранение `export()` обратно в IndexedDB (та же логика debounce 1000ms).
- Битый дамп → бэкап-ключ + новая база (как в десктопе).

### 2. Настройки
- `electron-store` → тонкая обёртка над `localStorage` с тем же интерфейсом get/set/getAll. Schema и дефолты переносятся 1:1 (провайдер уже `proxyapi`).

### 3. AI через Cloudflare Worker
- Worker принимает POST от браузера, проксирует на `https://api.proxyapi.ru/openai/v1/chat/completions`, **добавляет CORS-заголовки**. Ключ юзера (BYOK) идёт в `Authorization` насквозь, Worker его **не хранит**.
- Web-адаптер `ai.generateBatch/mnemonic` дёргает Worker вместо `openai` SDK (тот же промпт и парсинг из `gemini.service.ts` переносим в браузер/адаптер).
- Worker бесплатный (free tier 100k запросов/день).

### 4. TTS
- `tts.generateAudio` → `window.speechSynthesis` (Web Speech API). Определение языка (кириллица → ru, иначе en) переносим. Аудио не хранится — синтез на лету.

### 5. PWA
- `manifest.json` (имя, иконки, theme-color графитовый, display standalone).
- Service worker (Workbox или ручной): офлайн-кеш статики + WASM sql.js. Кнопка «Установить».

### 6. Сборка и деплой
- Отдельный webpack web-таргет (`target: web`, без electron). Точка входа — существующий renderer.
- Деплой: Cloudflare Pages (подключить к GitHub-репо, авто-сборка из ветки). Worker — отдельный деплой через `wrangler`.

### 7. Магнит-колоды (seed)
- Готовые колоды **Лексика A1, A2, B1** (по CEFR) — встроены как seed-данные, подгружаются при первом запуске пустой базы. Контент карточек подготовить отдельно (вне кода).

## Отложено в Фазу 2 (явно НЕ в v1)

- **Импорт из Anki** — текущий механизм через AnkiConnect (localhost HTTP) в браузере невозможен. Требует переписывания на парсинг `.apkg` (JSZip + sql.js). Отдельная задача.
- **Media-хранилище** (импортированное аудио) — нужно только вместе с Anki-импортом. В v1 произношение через Web Speech, файлы не хранятся.
- ElevenLabs / премиум-голоса, синк, подписки.

## Что нужно от Данька

- **Аккаунт Cloudflare** (бесплатный) — для деплоя Pages + Worker.
- **ProxyAPI-ключ** — для теста Worker-прокси и генерации (BYOK; в проде каждый юзер вводит свой).
- Контент **магнит-колод A1/A2/B1** — список слов (можно сгенерировать тем же AI позже).
- Домен `qard.app` — потом, старт на `qard.pages.dev`.

## Риски и оговорки

- **Persistence IndexedDB** — может стираться при ручной очистке данных сайта. Лечится: экспорт/импорт бэкапа БД (кнопка) + установка PWA. Заложить экспорт/импорт в v1.
- **BYOK-ключ в браузере** — хранится в localStorage юзера, уходит только на его ProxyAPI через Worker. Для бесплатного локального продукта приемлемо; managed-ключи — этап монетизации.
- **Worker free tier** — 100k запросов/день, с запасом на старте.
- **Web Speech качество голосов** — зависит от ОС/браузера. Приемлемо для v1, премиум-голоса позже.

## Критерии готовности v1

- Открыл `qard.pages.dev` → создал колоду → вставил список слов → AI сгенерил карточки (через Worker) → прошёл review (type-in, FSRS) → данные сохранились локально и пережили перезагрузку → услышал произношение → установил как PWA → выгрузил/загрузил бэкап.
