# Дизайн: GTM-стратегия запуска Qard (distribution-first)

**Дата:** 2026-06-18
**Статус:** одобрен (с правками от Данька)
**Связано:** память — `monetization-strategy`, `project-state`, `project-qard-repo`

## Контекст и решения этого захода

- **Трек:** раскрутка / go-to-market (выбран первым из трёх; продакшн и продуктовые фичи — следующими заходами).
- **Главная установка Данька:** distribution-first. Сначала рост **бесплатной** базы, монетизация (Pro ₽120/мес) — потом, когда аудитория тёплая.
- **Формат доставки:** **Web-PWA — главный канал**, Electron-сборка — вторичная опция. Обоснование: web-версия Qard стоит ≈$0 (sql.js = SQLite в браузере через WASM, данные локально у юзера в IndexedDB/OPFS, хостинг статики бесплатный). Опасение «web = платные серверы/БД» неверно для client-side PWA.
- **Архитектура готова к порту:** все Electron/Node-зависимости (`fs`, `path`, `electron-store`) сидят только в `src/main/**`; UI (`src/renderer/**`) — чистый React поверх IPC-моста. Порт = переписать нижний слой `main` под браузерные API, сохранив контракт preload. UI не трогается.
- **Канал привлечения:** build in public, Данёк готов быть лицом.
- **Стратегия запуска:** B+C — build in public как костяк + залпы по вехам поверх.

## 1. Цель и метрика успеха

- **Цель этапа:** рост бесплатной базы + тёплая аудитория под будущую монетизацию (не деньги).
- **North Star:** число людей, кто **создал колоду и сделал ≥1 повтор** (активация, не пустые заходы).
- **Промежуточные KPI:** подписчики TG-канала, переходы на `qard.app`, конверсия «зашёл → создал колоду», D7 retention.
- **Первый рубеж:** ~100 активированных юзеров + ~300–500 подписчиков канала.

## 2. Позиционирование и оффер

- **Слоган:** «Anki для нормальных людей».
- **Оффер:** *«Вставь текст или список слов → получи умную колоду → учи с телефона. Бесплатно, без установки, данные у тебя».*
- **3 кита против боли Anki:**
  1. **Ноль порога** — зашёл по ссылке, уже учишь (vs «скачай .apkg, поставь плагин»).
  2. **Умная генерация** — вставил слова, AI делает примеры/перевод/мнемонику.
  3. **Приватность** — данные локально, не на чужом сервере.

## 3. Технический фундамент запуска (что нужно в коде ради GTM)

- **Web-PWA порт** — главный блок. Переписать слой `src/main` под браузер: sql.js + IndexedDB/OPFS (хранилище), localStorage вместо electron-store, fetch для AI. UI (`renderer`) не меняется.
- **Лендинг** — одна страница: оффер, гифка «вставил → учишь», кнопки «Открыть Qard» / «Скачать десктоп». Хостинг бесплатный (Cloudflare/GitHub Pages).
- **Аналитика** — приватная (Plausible/Umami), для воронки.
- **Виральная петля** — **шаринг колод по ссылке** («моя колода 500 слов для IELTS → открыть в Qard»). Критичный для роста механизм.
- **Экспорт/импорт бэкапа** — лечит риск стирания браузерного хранилища + переносимость.
- **Стартовые колоды-магниты:** Лексика **A1, A2, B1** (по CEFR) — лёгкие, чтобы юзеру было что учить сразу.

## 4. Каналы и контент-план (B+C)

**Костяк (B) — build in public, Данёк ведёт лично:**
- **TG-канал** — основной дневник: что сделал, скрины фич, грабли, цифры, опросы. 2–3 поста/неделю.
- **YouTube** — длиннее: разборы, демо, «как я строю продукт».
- **Instagram Reels / Shorts** — короткие демо «вставил текст → колода за 5 сек», цепляющие фичи.
- Тон — честный инди-дневник, не реклама. Контент = побочный продукт работы.

**Залпы (C) — на вехах (статус: обдумывается Даньком):**
- **Хабр** — лонгрид «Как я делал Anki для нормальных людей на Electron+React» (tech-аудитория = первый сегмент).
- **vc.ru / Пикабу** — продуктовая история.
- **Product Hunt / аналоги** — на запуске web-беты.
- **Тематические TG/VK-сообщества** по английскому — польза, не спам.
- **Reddit** (r/Anki, r/languagelearning) — задел под глобал позже.

**Вехи-инфоповоды:** запуск web-беты → 100 юзеров → импорт из субтитров → 500 юзеров → мобилка.

## 5. Воронка

```
  ACQUISITION   контент (TG/YT/Reels) + залпы → лендинг qard.app
       │
  ACTIVATION    зашёл → создал колоду → 1-й повтор   ← North Star
       │
  RETENTION     PWA «установить» + streak + дневная цель (уже в коде!)
       │
  REFERRAL      шаринг колоды по ссылке → новый юзер
```

Удержание частично уже закрыто: streak 🔥, дневная цель, прогресс-хедер — сильный задел.

## 6. Последовательность (фазы)

0. **Сейчас:** завести TG-канал, первый пост build in public; собрать десктоп-сборку текущего проекта (чтобы Данёк пользовался уже сейчас). Аудитория копится параллельно порту.
1. **Фаза 1:** web-PWA порт + лендинг + аналитика → закрытая бета для подписчиков.
2. **Фаза 2:** публичный запуск web-беты + первый залп (Хабр). Шаринг колод включён.
3. **Фаза 3:** залпы по вехам, итерации по фидбеку, наполнение магнит-колодами (A1/A2/B1, IELTS, топ-3000).
4. **Фаза 4 (потом):** монетизация Pro, когда база тёплая.

## 7. Риски и оговорки

- **Дисциплина контента** — главный риск B. Решение: контент = побочный продукт работы.
- **Persistence браузера** — закрыто бэкапом + PWA-установкой.
- **Пустой продукт на старте** — магнит-колоды A1/A2/B1.
- **CORS для AI** — уже решено через ProxyAPI.
- **managed-генерация стоит денег за API** — бесплатная версия должна быть BYOK/локальная, иначе рост = убыток. Учесть на этапе монетизации.
- **Не распыляться на сегменты** — на старте только tech/RU-TG.
