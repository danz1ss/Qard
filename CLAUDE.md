# Настройки ассистента Claude (Режим максимальной эффективности)

## 1. Глобальные правила общения (Anti-Chatter)

- Отвечай максимально кратко и строго по делу.
- КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНЫ: приветствия, извинения, вежливости, слова-паразиты и фразы вроде "Конечно, я помогу", "Вот обновленный код", "Надеюсь, это решит проблему".
- Если я задаю вопрос — дай прямой ответ. Если прошу написать код — выведи только код.

## 2. Навигация и архитектура (Graphify First)

- Твой главный источник знаний об архитектуре системы — сгенерированный файл `graph.json` (или `graphify-out/graph.json`).
- ПРАВИЛО №1: При любом запросе на анализ, рефакторинг или добавление фичи СНАЧАЛА обращайся к графу.
- ЗАПРЕЩЕНО использовать команды сканирования директорий вслепую для поиска нужного файла.
- Читай полные исходники файлов (исходный код) ТОЛЬКО после того, как нашел нужный узел в графе и тебе критически важно увидеть точную реализацию конкретной функции.

## 3. Генерация кода и изменения (Strict Diff)

- Токены вывода дорогие. НИКОГДА не выводи файл целиком, если нужно изменить только несколько строк.
- Всегда выводи изменения в строгом формате `diff` (или используя Search & Replace блоки), показывая только изменяемые строки и 2-3 строки контекста вокруг них.

Пример ожидаемого вывода для изменений:

```diff
--- src/database/config.ts
+++ src/database/config.ts
@@ -12,3 +12,3 @@
 const dbTimeout = 5000;
-const maxConnections = 10;
+const maxConnections = 25;
 export const connect = () => {
```
